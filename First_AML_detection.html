Problem

The International Monetary Fund (IMF) estimates the annual amount laundered globally at $800 billion to $2 trillion. Similarly, according to UNODC estimates cited by FATF on their latest 2024-2025 report, up to USD 2 trillion is laundered globally every year — and global asset recovery efforts still fall well short of addressing the estimated hundreds of billions in criminal proceeds circulating in the financial system annually.

Following the same FAFT report, only 53% of countries update their national risk assessments every 2–3 years, 80% face data collection challenges, and 35% report coordination failures due to limited political commitment — creating persistent blind spots that criminals exploit.

Fraud is now the #2 predicate crime worldwide. The latest round of FATF mutual evaluations found that fraud is cited in 89% of country reports (156 countries), making it the second most common crime feeding money laundering after corruption — and financial fraud is described as “one of the fastest growing threats on a global scale. Moreover, studies from ACFE show a direct correlation between economic downswings and fraudulent activity, which may be particularly relevant in the current economic and geopolitcal climate.

There is also a fast-growing loophole on virtual assets. With VA increasingly misused as channels for financial crime, global compliance remains dangerously low: implementation of FATF standards is significantly lower in the virtual asset sector than in traditional finance, and many jurisdictions have yet to regulate crypto service providers at all.

Money laundering remains a relevant problem worldwide, with a worrying financial impact. Additionally, we should remember that money laundering always involves a predicate crime with a direct impact on people’s lives. Fighting money laundering is not only a financial issue, but also a social one.

This document focus on the detection of money laundering activities using the Anti Money Laundering Transaction Data (SAML-D) available on Kaggle.

The notebook also realies heavily on the AML Prediction Notebook by Gaurav Bhattacharya and following comments for enhancing its methodology and results.

Another imporant pattern to analyse when looking for money laundering activities may be the time of the day when the transactions are made. To acomplish this, we need to process the Time and Date variables which, on the raw dataset, are in string format.

Code 
# Converting date into datetime format
raw_df['Date'] = pd.to_datetime(raw_df['Date'])

# Extract month, day, and week
raw_df['Year'] = pd.to_datetime(raw_df['Date']).dt.year
raw_df['Month'] = pd.to_datetime(raw_df['Date']).dt.month
raw_df['Day'] = pd.to_datetime(raw_df['Date']).dt.day
raw_df['Week'] = raw_df['Date'].dt.isocalendar().week

# Extract hour from Time
raw_df['Hour'] = pd.to_datetime(raw_df['Time'], format='%H:%M:%S').dt.hour

# Reestablishing the ML_data dataframe with the new variables
ML_data = raw_df[raw_df['Is_laundering'] == 1]

# Hour analysis
hourly_transactions = ML_data.groupby('Hour').size()
average_hourly_transactions = hourly_transactions.mean()

# Plot hour analysis
plt.figure(figsize=(10, 6))
hourly_transactions.plot(kind='line')
plt.axhline(y=average_hourly_transactions, color='r', linestyle='--', label=f'Average Transactions ({average_hourly_transactions:.2f})')
plt.xlabel('Hour of the Day')
plt.ylabel('Number of Transactions')
plt.title('Number of Transactions Per Hour')
plt.legend()
plt.grid(True)
plt.show()

This graph doesn’t seem particularly useful, what if we add the clean transactions?

Code 
# Hour analysis - separate by laundering status
hourly_ml = raw_df[raw_df['Is_laundering'] == 1].groupby('Hour').size()
hourly_non_ml = raw_df[raw_df['Is_laundering'] == 0].groupby('Hour').size()

# Create figure with dual y-axes
fig, ax1 = plt.subplots(figsize=(12, 6))

# Plot money laundering transactions on left y-axis
color = 'tab:red'
ax1.set_xlabel('Hour of the Day')
ax1.set_ylabel('Money Laundering Transactions', color=color)
ax1.plot(hourly_ml.index, hourly_ml.values, color=color, marker='o', label='Money Laundering')
ax1.tick_params(axis='y', labelcolor=color)

# Create second y-axis for non-money laundering transactions
ax2 = ax1.twinx()
color = 'tab:blue'
ax2.set_ylabel('Non-Money Laundering Transactions', color=color)
ax2.plot(hourly_non_ml.index, hourly_non_ml.values, color=color, marker='s', label='Non-Money Laundering')
ax2.tick_params(axis='y', labelcolor=color)

# Add title and grid
plt.title('Hourly Transaction Patterns: Money Laundering vs. Non-Money Laundering')
fig.tight_layout()
plt.grid(True, alpha=0.3)
plt.show()

When working on fraud detection we usually expect to see a higher number of fraudulent transactions during non-business hours, mainly because of the lower monitoring and controls during those hours. However, in this dataset we can see that the number of money laundering transactions spikes along with the number of clean transactions, maybe mimicking the same patterns to avoid detection. However, interestingly we can see a deacresing trend on the number of money laundering transactions during as the day progresses, while the number of clean transactions remains relatively stable.
